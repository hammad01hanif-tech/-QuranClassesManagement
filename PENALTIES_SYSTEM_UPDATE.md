# ✅ تحديثات نظام الخصومات والسماح

## 📋 ملخص التحديثات

تم تحسين نظام الخصومات ليكون **ذكي واحترافي ومرتبط بشكل سلس** بين صفحة المعلم وصفحة الإدارة.

---

## 🎯 الميزات الجديدة

### 1️⃣ **نظام السماح الذكي (Pardon System)**

#### في صفحة الإدارة:
- ✅ عند النقر على أي خصمية في السجل الشهري، يظهر خياران:
  - **اعتماد الخصم** ✅: تثبيت الخصمية
  - **سماح (إلغاء الخصم)** 🙏: إلغاء الخصمية مع حفظ القيمة الأصلية

#### حفظ القيمة الأصلية:
- عند السماح، يتم حفظ القيمة في حقول جديدة:
  - `lateDeductionOriginal`
  - `earlyLeaveDeductionOriginal`
  - `absenceDeductionOriginal`
- يمكن إعادة اعتماد الخصمية لاحقاً واستعادة القيمة الأصلية

#### عرض السماح في السجل الشهري:
```
خصمية التأخير: ̶1̶5̶ ̶ر̶ي̶ا̶ل̶ سماح
خصمية الخروج المبكر: ̶1̶0̶ ̶ر̶ي̶ا̶ل̶ سماح
الغياب: غائب 🔴
         ̶5̶0̶ ̶ر̶ي̶ا̶ل̶ سماح
```

---

### 2️⃣ **التحديث التلقائي في بطاقة الخصومات**

#### في صفحة المعلم (الاحصائية الشهرية):
- ✅ **تحديث فوري** عند أي تغيير في السجل الشهري
- ✅ **استثناء الخصومات المسموح بها** من المجموع الكلي
- ✅ **عرض تفصيلي** للخصومات:
  - عدد حالات التأخير
  - عدد حالات الخروج المبكر
  - عدد حالات الغياب

#### نظام الاستماع المزدوج:
1. **setupPenaltiesListener**: يعمل طوال الوقت في جميع التبويبات
2. **teacherAttendanceListener**: ينشط عند فتح تبويب "الحضور والراتب"

---

### 3️⃣ **أنواع الخصومات المدعومة**

| النوع | الرمز | يظهر في السجل | يظهر في البطاقة | يدعم السماح |
|------|------|--------------|----------------|-------------|
| التأخير | 🕐 | ✅ | ✅ | ✅ |
| الخروج المبكر | 🚪 | ✅ | ✅ | ✅ |
| الغياب | 🔴 | ✅ | ✅ | ✅ |

---

### 4️⃣ **التكامل مع إعدادات المعلم**

#### فحص التفعيل/التعطيل:
- ✅ قبل حساب أي خصمية، يتم التحقق من:
  - `latePenalty.enabled`
  - `earlyLeavePenalty.enabled`
  - `absencePenalty.enabled`
- ✅ إذا كانت معطلة، لا يتم حساب الخصمية

#### زر تفعيل/تعطيل الخروج المبكر:
- ✅ موجود في صفحة الإدارة → إعدادات المعلم
- ✅ مرتبط بشكل مباشر مع حسابات الخصومات

## 🔧 التفاصيل التقنية

### حقول Firestore الجديدة:
```javascript
{
  // القيم الحالية
  lateDeduction: 15,
  earlyLeaveDeduction: 10,
  absenceDeduction: 50,
  
  // حالة الاعتماد/السماح
  lateApprovalStatus: 'pardoned' | 'approved' | undefined,
  earlyLeaveApprovalStatus: 'pardoned' | 'approved' | undefined,
  absenceApprovalStatus: 'pardoned' | 'approved' | undefined,
  
  // القيم الأصلية (قبل السماح)
  lateDeductionOriginal: 15,
  earlyLeaveDeductionOriginal: 10,
  absenceDeductionOriginal: 50
}
```

### دوال معدلة:
1. **updatePenaltyStatus()** في `admin.js`:
   - حفظ القيمة الأصلية قبل السماح
   - استعادة القيمة عند إعادة الاعتماد
   - console.log محسّنة للتتبع

2. **getMonthlyPenalties()** في `teacher.js`:
   - استثناء الخصومات المسموح بها (`approvalStatus === 'pardoned'`)
   - حساب دقيق للمجموع الكلي

3. **setupPenaltiesListener()** في `teacher.js`:
   - فحص حالة السماح قبل الحساب
   - تحديث `window.currentPenaltiesData`
   - تحديث تلقائي للبطاقة

4. **openAttendanceRecordModal()** في `teacher.js`:
   - عرض القيمة الأصلية عند السماح
   - onclick handlers للخصومات المسموح بها
   - عرض خصمية الغياب في خلية الحضور

## 📊 كيفية الاستخدام

### للإدارة:

1. **فتح السجل الشهري**:
   - الإدارة → حضور الموظفين → اختر المعلم → اختر الشهر

2. **عرض/تعديل الخصمية**:
   - انقر على أي خصمية (تأخير/خروج مبكر)
   - أو انقر على صف الغياب

3. **اختيار الإجراء**:
   - **اعتماد**: تثبيت الخصمية ✅
   - **سماح**: إلغاء الخصمية 🙏

4. **إعادة الاعتماد**:
   - انقر على الخصمية المسموح بها
   - اختر "اعتماد" لاستعادة القيمة الأصلية

### للمعلم:

1. **عرض الخصومات**:
   - تبويب "الحضور والراتب"
   - البطاقة تُحدّث تلقائياً

2. **التفاصيل**:
   - انقر على بطاقة "الخصومات الحالية"
   - عرض جميع الخصومات بالتفصيل

## 🧪 الاختبار

### سيناريو 1: السماح للتأخير
```
1. افتح السجل الشهري من صفحة الإدارة
2. انقر على خصمية تأخير (مثلاً 15 ريال)
3. اختر "سماح"
4. تحقق: الخصمية تصبح 0 في Firestore
5. تحقق: القيمة الأصلية محفوظة في lateDeductionOriginal
6. تحقق: بطاقة المعلم تتحدث تلقائياً
```

### سيناريو 2: إعادة اعتماد خصمية
```
1. انقر على خصمية مسموح بها (مشطوبة + "سماح")
2. اختر "اعتماد"
3. تحقق: القيمة الأصلية تعود إلى lateDeduction
4. تحقق: lateDeductionOriginal يصبح null
5. تحقق: بطاقة المعلم تتحدث تلقائياً
```

### سيناريو 3: تعطيل خصمية من الإعدادات
```
1. الإدارة → إعدادات المعلم
2. عطّل "خصمية الخروج المبكر"
3. عدّل حضور المعلم بخروج مبكر
4. تحقق: Console يظهر "⚠️ Early leave penalty is DISABLED"
5. تحقق: لا تحسب خصمية
```
